This method maps the given permission value to its corresponding numeric representation based on the Bluetooth host stack type (EMB or AW). The function iterates through each permission in the Att_permissions_t enum and checks if the permission bit is set in the provided PermissionValue. It then accumulates the corresponding mapped value from the dictionary to form the final result.

Inquiry(local_address=None, mode=2, duration=5000, advertisement_interval=250, advertisement_window=250, advertisement=1, scan_response=0, check_adv=None, check_rsp=None, timestamp_enable=False)

This method starts an inquiry process in the network and it process the incoming advertisements and response packets.

Parameters:

  • local_address (DB_ADDR) – A valid pointer to the Bluetooth device address of the remote device.

  • mode (int) – The desired inquiry mode (LIMITED or GENERAL).

  • duration (int) – the duration, in millisecond, of the inquiry procedure.

  • advertisement_interval (int) – Desired scanning interval to scan the devices.

  • advertisement_window (int) – Desired scanning window to scan the response of the devices.

  • advertisement (int) – Enable/Disable advertisement scanning. Function checks if the advertisement count is higher than the advertisement value. The default value is 1.

  • scan_response (int) – Enable/Disable the response scanning. Function checks if the response count is higher than scan_response value. The default value is 1.

  • check_adv (int) – check for specific data inside of the advertisement packet. If default value (NONE), the process does not filter the packets.

  • check_rsp (int) – check for specific data inside of the response packet. If default value (NONE), the process does not filter the packets.

  • timestamp_enable (boolean) – Enable/Disable timestamp capture when an observation is received.

Returns::

Status (int): return the Gap_status_t value.

  • addr_type (int) – The type of address of the address added to the list. Either BLEADDRESS_PUBLIC or BLEADDRESS_RANDOM.

  • scan_mode – Scan mode for the scanning : None, Limited or General

  • scan_type – Scan type: Passive or Active

  • duplicateFiltering (int) – enable or disable duplicate filtering capability.

  • duration (int) – timeout to stop the observation process.

  • advertisement_interval (int) – Desired scanning interval to scan the devices when observe.

  • advertisement_window (int) – Desired scanning window to scan the devices when observe.

  • advertisement (int) – Enable advertisement checks. Function checks if the observations count is higher than the advertisement value. The default value is 1.

  • check_obs (int) – check for specific data inside of the response packet. If default value (NONE), the process does not filter the packets.

  • timestamp_enable (boolean) – Enable/Disable timestamp capture when an observation is received.

  • whitelist (boolean) – Enable/Disable to add(before observation) and remove(after observation) the device in the whitelist.

  • callback (fuction) – Callback funtion to call after start the observation process. This callback function should not require any parameter.

  • timeCallbackSteps (int) – Required time to execute the callback function, this time will be added to duration time to get the total time observation when callback function is executed.
