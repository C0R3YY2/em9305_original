blengine.core.config module

The Config module

This module bind a ConfigObj object to the “config” object. ConfigObj extends the INI format please read the doc.

It offer to function to load configuration file and to validate config tree.

To ease the development with git, please add _local. to your gitignore: if there is a file under the form of <filename>_local.<extension> it will also load and merge it after the original.

To load a config file and merge it in the config object :

from blengine.core import config

config.load() # load config.cfg file in the module root folder

myconfig = config.load("myconfig.ini") # load myconfig.ini from the current

# parse and load the config.cfg content in the current script folder
config.load("config.cfg", __file__)

to use the global config object

from core.config import config

config["section"]["key"]
config.get("Key")

the ConfigObj can contain any level of indentation, given the config: (this can be indented)

[section]
    [[subsection]]
        value=1

access value with

config["section"]["subsection"]["value"]

To use more speciffically the config object see ConfigObj module documentation.

To validate a config object, refer to the ConfigObj docuemntation. IF there is any validation error, an exception will be raised.

Example from transport:

form blengine.core import config

config.load("config.ini")

cfg = config.config["serial_port"]

default_values =
port = string(min=1, max=30)
baudrate = integer(min=0, max=1000000, default=115200)
parity = boolean(min=0, max=100, default=None)
timeout = float(min=0, max=100, default=1)

cfg = config.validate(cfg, default_values.split())
ser = serial.Serial()

ser.baudrate = cfg["baudrate"]

class blengine.core.packets.GenericPacket(name='', data=b'')

Bases: FragmentList

Base class for a packet

Unique ID: a GenericPacket is identified by a uid, this uid have to match the beginning of the encoded packet because it’s used to identify one.

Wrappers: This class contain special “wrapper” fields to define either the inner-packet defined by the remaining data with self.decode_wrapper that is the class name of the encapsualted packet. or if the packet itself have to be encapsulated into another packet with self.encode_wrapper which reference the encapsulated packet to be encoded with

With this wrappers a packet can be dynamically encapsulated into another one and also dynamically decoded to another packet depending of it’s content.

For example the HCI Reset encode wrapper reference the HCI_CMD packet. And the Reserved_Vendor_Event as decode_wrapper set to “HCIVendorEventPacket”.

the wrap() and unwrap() methods take care of this

blengine.core.transport module

The Transport module

This module provide base classes for a transport layer.

The Transport class inherit from IOService.

If you want to implement your own, you have to reimplement the following functions:

  • open(self)

    will be called when the transport is started, use it to initialize wathever you need to

  • close(self)

    will be called when transport is stopped. here you have to free allocated and close sockets

  • read(self)

    this function will be called inside the service loop as the producer function. it has to return something

  • write(self, data)

    this function will write whatever data is and return the length if wrritten, otherwise “0”
