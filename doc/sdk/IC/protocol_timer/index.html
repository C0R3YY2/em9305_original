Protocol Timer Driver

Overview

The protocol timer driver is used by the BLE stack and should not be accessed directly.

The API is shown here for informational purposes only.

Warning

Using this API incorrectly can have an impact on BLE operations.

static void ProtTimer_SetFullValue(uint32_t fullValue)

Set full value of Protocol Timer.

It sets full value of Protocol Timer. Once full value is reached interrupt is generated and Protocol Timer either stops to count or it reloads to zero and then it continues. Protocol Timer counts from 0 to fullVal (number of ticks between two interrupts is equal to fullValue+1). If fullValue is set to -1 Protocol Timer counts up to 0xFFFFFFFF and then it continues from zero.

Parameters:

fullValue – Full value to be set.

void ProtTimer_SetDivider(uint8_t divFactor)

Set divider factor.

Protocol timer is running on 24MHz/divFactor except value 0. Divider factor 0 is interpreted as 256. With divider factor of 1 Protocol timer is running at full speed (24MHz). With divider factor of 255 Protocol timer is running at 24MHz/255. With divider factor of 0 (256) Protocol timer is running at 24MHz/256.

Note

Divider factor shall be changed only if Protocol timer is not running. No check of running Protocol timer is done. Valid range is not checked, bigger values are presented as divFactor mod 256.

Parameters:

divFactor – Divider factor, allowed range <0:255>

void ProtTimer_SetConfiguration(bool autoReload, bool syncStart)

Set Protocol Timer configuration.

It enables/disables Auto Reload mode and Synchronous Start/Stop. In Auto Reload mode Protocol Timer continues in counting from zero after reaching Full Value. Synchronous Start/Stop ensures that timer is started/sopped synchronously with low frequency clock. This ensures that timing is correctly maintained in sleep modes. This mode is required when used together with Sleep Timer.

Note

After changing Synchronous Start/Stop state SW needs to wait for at least 4 clock cycles (4 NOP instructions) before starting/stopping Protocol Timer.

Parameters:

  • autoReload – Enable for Auto Reload mode.

  • syncStart – Enable for synchronous start/stop.

void ProtTimer_Load(uint32_t loadValue)

Load Protocol Timer.

It preloads Protocol Timer with specified value. After preloading Protocol Timer value is equal to load value and Protocol Timer continues from this loaded value.

Note

Protocol Timer can be preloaded even when running.

Parameters:

loadValue – Value to be loaded into Protocol Timer.

void ProtTimer_Adjust(uint32_t adjustValue)

Adjust Protocol Timer value.

It adjusts Protocol Timer value with specified value. After adjusting Protocol Timer value is equal to old_value + adjustValue. Adjustment can be done in both dirrection (add/subtract).

Note

Protocol Timer can be adjusted even when running.

Parameters:

adjustValue – Value to be added into Protocol Timer current value.

ProtTimer_EventTimeResult ProtTimer_GetTimeToFirstEvent(SystemTime_t *pTimeToFirstEvent, SystemTime_t *pDeltaToFirstEvent, SystemTime_t *pQueryStart)

Get the time to the first event that is scheduled in the future.

Modules can register to the protocol timer module with the function ProtTimer_RegisterEventTimeCallback. To get the time to the first event in the future, the registered “event time callbacks” are questioned one after the other.

Parameters:

  • pTimeToFirstEvent – [out] Timestamp of the next event.

  • pDeltaToFirstEvent – [out] Delta time to the next event.

  • pQueryStart – [out] Reference timestamp for pDeltaToFirstEvent .

Return values:

  • PT_EVENT_TIME_STAMP_VALID – The time to the first event is returned in the parameters pTimeToFirstEvent , pDeltaToFirstEvent and pQueryStart .

  • PT_EVENT_TIME_INFINITE_TIME – No event are planned in the future.

  • PT_EVENT_TIME_SLEEP_FORBIDDEN – At least one module has an event in the future but the time of that event is not known.
