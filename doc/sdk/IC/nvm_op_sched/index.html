NVM operation scheduler

Overview

This library is used to schedule write and erase operations in the EM9305 FLASH memory (NVM) when the system is idle (e.g., no ongoing radio operation).

API

group nvm_op_sched

Library to schedule the NVM operations.

This module contains all the code required to schedule the NVM operations.

Defines

NVM_OP_SCHED_QUEUE_SIZE (8u)

The maximum number of scheduled NVM operation that can be queued.

int32_t NVMOpSched_ScheduleErase(uint32_t addr, uint32_t length, NVMOpSched_DriverCallback_t operationCb, void *pUserData)

Schedule a NVM-Main or NMV-Info erase page(s).

Note

The maximum number of scheduled NVM operation is NVM_OP_SCHED_QUEUE_SIZE.

Parameters:

  • addr – Start address, must be NVM page size aligned.

  • length – Number of bytes to erase, must be NVM page size aligned.

  • operationCb – Callback that will be called when the operation is completed.

  • pUserData – User data that is passed as a parameter when the callback is invoked.

Returns:

Error code, see errno.h.

int32_t NVMOpSched_ScheduleWrite(uint32_t addr, uint32_t length, uint32_t *pWriteData, NVMOpSched_DriverCallback_t operationCb, void *pUserData)

Schedule a write to NVM-Main or NMV-Info.

Note

The maximum number of scheduled NVM operation is NVM_OP_SCHED_QUEUE_SIZE.

Parameters:

  • addr – Start address, must be NVM word size aligned.

  • length – Number of bytes to write, must be NVM word size aligned.

  • pWriteData – Pointer to the data to write to NVM.

  • operationCb – Callback that will be called when the operation is completed.

  • pUserData – User data that is passed as a parameter when the callback is invoked.

Returns:

Error code, see errno.h.

void NVMOpSched_TryToPerformNvmOp(void)

Try to perform the NVM operations that were scheduled.

This function is intended to be called when the system is executing its idle task. The NVM scheduler will query all modules that have registered a function with the protocol timer module using ProtTimer_RegisterEventTimeCallback.

Note

When NVM operations are waiting to be executed, the NVM operation scheduler will prohibit the CPU from going to sleep and halting.
